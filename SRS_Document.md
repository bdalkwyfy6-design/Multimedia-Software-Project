وثيقة مواصفات متطلبات البرمجيات (SRS) - منصة عرض أعمال الطلبة التفاعلية

1. بيان المشكلة (Problem Statement)

يواجه طلبة تخصصات التصميم والإعلام تحدياً كبيراً في عرض مشاريعهم الإبداعية (فيديو، صور، موشن جرافيك) بطريقة احترافية ومستدامة. فغالباً ما تضيع هذه الجهود بعد تسليمها للتقييم، ولا يوجد معرض رقمي دائم يوثق إنجازاتهم أو يتيح لهم الحصول على تغذية راجعة من المشرفين والجمهور. يهدف هذا المشروع إلى بناء منصة تفاعلية تعمل كمعرض دائم لأعمال الطلبة، مما يسهل عليهم بناء هويتهم المهنية وربطهم بسوق العمل.




2. وثيقة مواصفات متطلبات البرمجيات (SRS Document)

2.1 المقدمة (Introduction)

نظام "منصة عرض أعمال الطلبة" هو تطبيق ويب مخصص لتوثيق وعرض المشاريع الإبداعية لطلبة الوسائط المتعددة. يهدف النظام إلى خلق بيئة تفاعلية تجمع بين الطلبة والمشرفين والزوار لتبادل الخبرات والتقييمات.

2.2 الوصف العام للنظام (Overall Description)

النظام عبارة عن منصة ويب تتيح للطلبة رفع ملفات الوسائط المتعددة الثقيلة، وتوفر واجهات مخصصة للمشرفين للتقييم، وللزوار للتصفح والتفاعل. يعتمد النظام على بنية تحتية تدعم تخزين ومعالجة الفيديو والصور عالية الدقة.

2.3 المتطلبات الوظيفية (Functional Requirements)

أ. متطلبات الطالب (Student Requirements):

•
إدارة الحساب الشخصي: إنشاء حساب طالب جديد، تسجيل الدخول، وتعديل الملف الشخصي.

•
إدارة المشاريع: رفع مشاريع جديدة (فيديو، صور، موشن جرافيك) مع إضافة عنوان ووصف وتصنيفات، وتعديل أو حذف المشاريع الخاصة به.

•
التفاعل: مشاهدة تقييمات وتعليقات المشرفين والزوار على مشاريعه والرد عليها.

ب. متطلبات المشرف (Supervisor Requirements):

•
إدارة ومراجعة المشاريع: تصفح جميع مشاريع الطلبة، تقييم المشاريع (نظام نجوم/درجات)، وكتابة تعليقات تفصيلية.

•
إدارة الطلبة: عرض قائمة بالطلبة المسجلين والبحث عن طالب معين وعرض ملفه الشخصي.

ج. متطلبات الزائر (Visitor Requirements):

•
تصفح المشاريع: تصفح المشاريع المعروضة حسب التصنيف أو الطالب، ومشاهدة المشاريع (فيديو، صور، موشن جرافيك).

•
التفاعل: تقييم المشاريع وكتابة تعليقات عامة، والبحث عن مشاريع أو طلبة محددين.

2.4 المتطلبات غير الوظيفية (Non-functional Requirements)

•
الأداء (Performance): سرعة تحميل الواجهة ومعاينة الوسائط في أقل من ثانيتين، واستجابة عمليات البحث في أقل من ثانية.

•
الأمان (Security): تشفير بيانات المستخدمين والملفات المرفوعة، وحماية الاتصالات باستخدام بروتوكول HTTPS.

•
التوافر (Availability): ضمان توفر النظام بنسبة 99.9% مع وجود نسخ احتياطي دوري للبيانات.

•
سهولة الاستخدام (Usability): واجهة مستخدم بديهية وجذابة تدعم اللغتين العربية والإنجليزية، وتصميم متجاوب يعمل على جميع الأجهزة.

2.5 متطلبات الواجهات الخارجية (External Interface Requirements)

•
واجهة المستخدم (GUI): واجهة ويب متجاوبة (Responsive Web Interface) متوافقة مع المتصفحات الحديثة.

•
واجهات برمجية التطبيقات (APIs): التكامل مع خدمات التخزين السحابي (مثل Amazon S3) لتخزين ملفات الوسائط الكبيرة.

2.6 قيود التصميم والتنفيذ (Design and Implementation Constraints)

•
استخدام لغة Java (مع إطار عمل Spring Boot) لتطوير الواجهة الخلفية لضمان الاستقرار وقابلية التوسع.

•
استخدام إطار عمل JavaScript حديث (مثل React) لبناء الواجهة الأمامية التفاعلية.

•
الالتزام بمعايير أمن البيانات العالمية وأفضل الممارسات في تطوير الويب الآمن.

2.7 التعريفات والمصطلحات (Glossary)

•
Portfolio (محفظة الأعمال): مجموعة من أعمال الطالب التي تعرض مهاراته وإنجازاته.

•
SRS: وثيقة مواصفات متطلبات البرمجيات.

•
Motional Graphics: رسوم متحركة رقمية تجمع بين الرسوم التوضيحية والفيديو والموسيقى.

•
Cloud Storage: تخزين البيانات على خوادم بعيدة عبر الإنترنت.
